## 1. 파일시스템 구조 (FHS: Filesystem Hierarchy Standard)

리눅스는 계층적 트리 구조를 가지며, 모든 하드웨어 장치와 디렉토리를 파일로 관리합니다.

| 디렉토리 | 설명 |
| :--- | :--- |
| **/** | **루트(Root):** 최상위 디렉토리. 모든 경로의 시작점 |
| **/bin** | 기본적인 사용자 명령어 (ls, cp, mv, cat 등) |
| **/etc** | **시스템 설정 파일** (계정 정보, 네트워크 설정, 서비스 설정 등) |
| **/home** | 일반 사용자들의 개인 홈 디렉토리 |
| **/root** | 시스템 관리자(root) 전용 홈 디렉토리 |
| **/var** | 로그(log), 메일, 데이터베이스 등 내용이 가변적인 데이터 저장 |
| **/usr** | 사용자 응용 프로그램, 라이브러리 및 매뉴얼 파일 |
| **/dev** | 시스템의 물리적 장치들을 파일 형태로 관리 (HDD, 키보드 등) |
| **/proc** | 실행 중인 프로세스 및 커널 정보를 담은 가상 파일 시스템 |

---

## 2. 리눅스 부팅 4단계

1.  **BIOS (Basic Input/Output System)**
    * 하드웨어 무결성 체크(POST)를 수행합니다.
    * 부팅 매체(HDD/SSD 등)를 찾아 첫 번째 섹터의 MBR을 읽습니다.
2.  **MBR (Master Boot Record)**
    * 부팅 장치의 첫 512바이트 영역입니다.
    * 부트 로더(GRUB)가 위치한 실행 정보를 담고 있습니다.
3.  **GRUB (Grand Unified Bootloader)**
    * 리눅스 전용 부트 로더로, 부팅할 커널을 선택하거나 옵션을 설정합니다.
    * 선택된 커널을 메모리에 로드합니다.
4.  **Kernel (커널)**
    * 시스템 자원을 초기화하고 제어권을 갖습니다.
    * 마지막에 `systemd`(또는 `init`) 프로세스를 실행하여 시스템 서비스를 시작합니다.

---

## 3. Run Level (런 레벨)

시스템이 구동되는 환경(모드)을 숫자로 구분한 설정입니다.

* **0 (Halt):** 시스템 종료 모드
* **1 (Single-user):** 시스템 복구 및 관리 모드 (네트워크 미사용)
* **3 (Multi-user, Text):** CLI 기반 다중 사용자 모드 (서버 환경 표준)
* **5 (Multi-user, GUI):** 그래픽(X-Window) 기반 다중 사용자 모드
* **6 (Reboot):** 시스템 재시작 모드

---

## 4. init + [No.] 명령어

현재 시스템의 런 레벨을 즉시 변경할 때 사용하는 명령어입니다.

* `init 0`: 시스템 종료 (Power Off)
* `init 6`: 시스템 재부팅 (Reboot)
* `init 3`: 텍스트 모드(CLI)로 즉시 전환
* `init 5`: 그래픽 모드(GUI)로 즉시 전환

## 5. 파일 내용 확인, 파이프, 리다이렉션

### 1) 파일 내용 확인 명령어
* `cat [파일명]`: 파일 전체 내용 출력
* `less [파일명]`: 페이지 단위로 확인 (스크롤 가능, `q`로 종료)
* `head -n [숫자] [파일명]`: 파일 상단부터 지정한 줄 수만큼 출력
* `tail -n [숫자] [파일명]`: 파일 하단부터 지정한 줄 수만큼 출력 (`-f` 옵션으로 실시간 로그 모니터링 가능)

### 2) 파이프 (Pipe, `|`)
* 앞 명령어의 **표준 출력**을 뒤 명령어의 **표준 입력**으로 넘겨줍니다.
* 예: `ps -ef | grep "httpd"` (실행 중인 프로세스 중 httpd 단어가 포함된 것만 필터링)

### 3) 리다이렉션 (Redirection, `>`, `>>`)
* 명령어의 출력 방향을 파일로 변경합니다.
* **덮어쓰기 (`>`)**: 파일이 없으면 생성하고, 있으면 기존 내용을 지우고 새로 씁니다.
    * 예: `ls > list.txt`
* **추가하기 (`>>`)**: 기존 내용을 보존하면서 파일의 끝에 새로운 내용을 덧붙입니다.
    * 예: `echo "New Log" >> system.log`
* **에러 추가 (`2>>`)**: 표준 에러 메시지만 지정된 파일 끝에 기록합니다.
