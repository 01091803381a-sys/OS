## 1. 파일시스템 구조 (FHS: Filesystem Hierarchy Standard)

리눅스는 계층적 트리 구조를 가지며, 모든 하드웨어 장치와 디렉토리를 파일로 관리합니다.

| 디렉토리 | 설명 |
| :--- | :--- |
| **/** | **루트(Root):** 최상위 디렉토리. 모든 경로의 시작점 |
| **/bin** | 기본적인 사용자 명령어 (ls, cp, mv, cat 등) |
| **/etc** | **시스템 설정 파일** (계정 정보, 네트워크 설정, 서비스 설정 등) |
| **/home** | 일반 사용자들의 개인 홈 디렉토리 |
| **/root** | 시스템 관리자(root) 전용 홈 디렉토리 |
| **/var** | 로그(log), 메일, 데이터베이스 등 내용이 가변적인 데이터 저장 |
| **/usr** | 사용자 응용 프로그램, 라이브러리 및 매뉴얼 파일 |
| **/dev** | 시스템의 물리적 장치들을 파일 형태로 관리 (HDD, 키보드 등) |
| **/proc** | 실행 중인 프로세스 및 커널 정보를 담은 가상 파일 시스템 |

---

## 2. 리눅스 부팅 4단계

1.  **BIOS (Basic Input/Output System)**
    * 하드웨어 무결성 체크(POST)를 수행합니다.
    * 부팅 매체(HDD/SSD 등)를 찾아 첫 번째 섹터의 MBR을 읽습니다.
2.  **MBR (Master Boot Record)**
    * 부팅 장치의 첫 512바이트 영역입니다.
    * 부트 로더(GRUB)가 위치한 실행 정보를 담고 있습니다.
3.  **GRUB (Grand Unified Bootloader)**
    * 리눅스 전용 부트 로더로, 부팅할 커널을 선택하거나 옵션을 설정합니다.
    * 선택된 커널을 메모리에 로드합니다.
4.  **Kernel (커널)**
    * 시스템 자원을 초기화하고 제어권을 갖습니다.
    * 마지막에 `systemd`(또는 `init`) 프로세스를 실행하여 시스템 서비스를 시작합니다.

---

## 3. Run Level (런 레벨)

시스템이 구동되는 환경(모드)을 숫자로 구분한 설정입니다.

* **0 (Halt):** 시스템 종료 모드
* **1 (Single-user):** 시스템 복구 및 관리 모드 (네트워크 미사용)
* **3 (Multi-user, Text):** CLI 기반 다중 사용자 모드 (서버 환경 표준)
* **5 (Multi-user, GUI):** 그래픽(X-Window) 기반 다중 사용자 모드
* **6 (Reboot):** 시스템 재시작 모드

---

## 4. init + [No.] 명령어

현재 시스템의 런 레벨을 즉시 변경할 때 사용하는 명령어입니다.

* `init 0`: 시스템 종료 (Power Off)
* `init 6`: 시스템 재부팅 (Reboot)
* `init 3`: 텍스트 모드(CLI)로 즉시 전환
* `init 5`: 그래픽 모드(GUI)로 즉시 전환

## 5. 파일 내용 확인, 파이프, 리다이렉션

### 1) 파일 내용 확인 명령어
* `cat [파일명]`: 파일 전체 내용 출력
* `less [파일명]`: 페이지 단위로 확인 (스크롤 가능, `q`로 종료)
* `head -n [숫자] [파일명]`: 파일 상단부터 지정한 줄 수만큼 출력
* `tail -n [숫자] [파일명]`: 파일 하단부터 지정한 줄 수만큼 출력 (`-f` 옵션으로 실시간 로그 모니터링 가능)

### 2) 파이프 (Pipe, `|`)
* 앞 명령어의 **표준 출력**을 뒤 명령어의 **표준 입력**으로 넘겨줍니다.
* 예: `ps -ef | grep "httpd"` (실행 중인 프로세스 중 httpd 단어가 포함된 것만 필터링)

### 3) 리다이렉션 (Redirection, `>`, `>>`)
* 명령어의 출력 방향을 파일로 변경합니다.
* **덮어쓰기 (`>`)**: 파일이 없으면 생성하고, 있으면 기존 내용을 지우고 새로 씁니다.
    * 예: `ls > list.txt`
* **추가하기 (`>>`)**: 기존 내용을 보존하면서 파일의 끝에 새로운 내용을 덧붙입니다.
    * 예: `echo "New Log" >> system.log`
* **에러 추가 (`2>>`)**: 표준 에러 메시지만 지정된 파일 끝에 기록합니다.
## 1. 아이노드 (inode)

아이노드는 리눅스 파일 시스템에서 파일을 식별하고 관리하는 **'파일의 신분증'**입니다.

* **역할:** 파일 이름은 사용자가 보기 편하게 만든 별명일 뿐, 시스템은 숫자로 된 아이노드 번호를 통해 데이터에 접근합니다.
* **저장 정보:** 파일 유형, 권한(rwx), 소유주 및 그룹, 파일 크기, 시간 정보(mtime, ctime, atime), 데이터 블록의 위치.
* **특징:** 파일 이름 자체는 아이노드에 저장되지 않고 디렉토리 엔트리에 저장됩니다.

## 2. 하드 링크 vs 심볼릭 링크

| 구분 | 하드 링크 (Hard Link) | 심볼릭 링크 (Symbolic Link) |
| :--- | :--- | :--- |
| **개념** | 원본과 **동일한 아이노드**를 공유 | 원본의 **경로명**을 담고 있는 별도 파일 |
| **아이노드** | 원본과 아이노드 번호가 같음 | 원본과 아이노드 번호가 다름 |
| **원본 삭제** | 데이터 유지 (링크 수가 0이 될 때까지) | 연결 끊김 (Dangling Link 발생) |
| **제약 사항** | 다른 파티션 생성 불가, 디렉토리 불가 | 파티션 제한 없음, 디렉토리 생성 가능 |
| **명령어** | `ln [원본] [링크]` | `ln -s [원본] [링크]` |

## 3. chmod (권한 설정)

파일이나 디렉토리에 대한 **읽기(r), 쓰기(w), 실행(x)** 권한을 설정합니다.

### 1) 숫자(8진수) 모드
권한 값을 더해 3자리 숫자로 표현합니다. (**r=4, w=2, x=1**)
* `755`: 소유자(rwx), 그룹(rx), 기타(rx)
* `644`: 소유자(rw), 그룹(r), 기타(r)
* `700`: 소유자(rwx), 나머지 권한 없음

### 2) 심볼릭 모드
대상과 연산자를 조합하여 설정합니다. (**u:소유자, g:그룹, o:기타, a:전체**)
* `chmod +x [파일명]`: 모두에게 실행 권한 추가
* `chmod g-w [파일명]`: 그룹의 쓰기 권한 제거
* `chmod u=rwx [파일명]`: 소유자에게 모든 권한 지정

## 4. Sticky Bit (스티키 비트)

공용 디렉토리에서 사용자의 파일을 보호하기 위한 특수 권한입니다.

* **용도:** `/tmp`와 같이 누구나 파일을 만들 수 있는 곳에서 **자신이 만든 파일은 자신과 root만 삭제**할 수 있게 제한합니다.
* **표시:** `ls -l` 시 Other 권한의 실행 자리에 **`t`**로 표시됩니다.
* **설정 방법:**
    * `chmod 1777 [디렉토리명]` (숫자 모드)
    * `chmod +t [디렉토리명]` (심볼릭 모드)

---

## 5. 소유권 변경 (chown)

파일이나 디렉토리의 소유자와 그룹을 변경합니다.

* **기본형:** `sudo chown [사용자]:[그룹] [대상]`
* **주요 옵션:**
    * `-R`: 하위 디렉토리와 파일까지 한꺼번에 변경 (Recursive)
* **예시:** `sudo chown -R student:devteam project/`
## 6. 터미널 기호 및 환경 요약

리눅스 CLI 환경에서 자주 접하게 되는 특수 기호와 프롬프트의 의미입니다.

### 1) 사용자 권한 및 프롬프트
터미널 입력창 맨 앞에 붙는 기호로 현재 사용자의 권한을 식별합니다.

| 기호 | 의미 | 권한 레벨 |
| :---: | :--- | :--- |
| **#** | **관리자 (Root)** | 시스템 전체 제어 가능 (주의 필요) |
| **$** | **일반 사용자** | 할당된 권한 내에서만 작업 가능 |

### 2) 경로 및 옵션 약어
* **`~` (Tilde)**: 현재 로그인한 사용자의 **홈 디렉토리(Home Directory)**를 의미합니다.
    * 예: `cd ~` (어디서든 홈으로 이동)
* **`-` (Short Option)**: 명령어 뒤에 붙는 단축 옵션 (예: `ls -l`)
* **`--` (Long Option)**: `--full`과 같이 단어 전체를 사용하는 옵션 (예: `ls --full-time`)

## 7. 파이프와 필터링 (| grep)

명령어의 실행 결과가 너무 많을 때, 원하는 내용만 골라내는 핵심 도구입니다.



### 1) 작동 원리
* **파이프 (`|`)**: 앞 명령어의 출력을 뒤 명령어의 입력으로 연결하는 통로입니다.
* **grep**: 입력받은 텍스트 중에서 특정 패턴(단어)이 포함된 줄만 찾아냅니다.

### 2) 주요 활용 사례
* **특정 프로세스 확인**: `ps -ef | grep [서비스명]`
    * 시스템에서 돌아가는 전체 목록 중 필요한 것만 추출
* **특정 파일 찾기**: `ls -al | grep ".conf"`
    * 현재 폴더 내 설정 파일(.conf)만 골라보기
* **로그 분석**: `cat access.log | grep "404"`
    * 웹 로그에서 '페이지 없음' 에러가 발생한 기록만 필터링

---

> **💡 Tip:** 관리자 권한(#)인 상태에서는 시스템을 파괴할 수 있는 명령어도 실행되므로, 평소에는 일반 사용자($) 권한으로 작업하다가 필요한 때만 `sudo`를 사용하는 것이 안전합니다.
